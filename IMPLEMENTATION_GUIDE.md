# Implementation Guide вҖ” Locked-in Design

Final design after your decisions. `BACKEND_PLAN.md` has the broader rationale; this doc is what to actually execute.

## Locked decisions

| Decision | Choice | Implication |
|---|---|---|
| **Database** | Self-hosted Postgres 16+ | You manage the host. Need backups, pgbouncer, monitoring. |
| **Raw scraper data** | `staging_raw_skus` table in same Postgres | One DB to manage; revisit S3 + Parquet only if staging grows past ~50M rows. |
| **Image hosting** | S3 (or S3-compatible: R2, Backblaze B2) | Bucket per env; CloudFront/Cloudflare in front for CDN. |
| **API stack** | Next.js (UI + SSR/ISR pages) + Express (API service) | Two processes; Express owns DB/business logic, Next.js calls Express. |
| **Auth** | None for MVP | All endpoints public; rate limit + bot detect carries the load. |
| **Rate limiting** | Redis sliding-window per IP + per route | `express-rate-limit` + `rate-limit-redis`. |
| **Bot detection** | Cloudflare in front + Turnstile on search | Cloudflare handles 95% of bots automatically; Turnstile challenge on suspicious behavior. |
| **Currency** | USD only | Store `price_cents INTEGER`; never floats. |
| **Tax** | State-based table at checkout | `tax_rates` (state, rate, effective_date). MVP: hardcoded 51 rows. Avalara/TaxJar later. |

**Critical postgresql.conf tunings (for 16GB RAM box):**
```
shared_buffers = 4GB
effective_cache_size = 12GB
work_mem = 32MB
maintenance_work_mem = 1GB
wal_compression = on

```

**pgbouncer (transaction pooling):**
```
pool_mode = transaction
default_pool_size = 25
max_client_conn = 1000
```

**Backups:**
- `pg_dump` daily вҶ’ upload to S3 (encrypted)
- WAL archiving for point-in-time recovery (`archive_command` вҶ’ S3)
- Restore test monthly (this is the part everyone forgets)

**Monitoring:**
- `postgres_exporter` + Grafana cloud free tier
- Healthchecks.io ping after each backup
- pg_stat_statements extension on for slow-query analysis

**When to add a read replica:** sustained Postgres CPU > 60%, or read latency > 200ms p95.

## Rate limiting + bot detection design

**Layer 1 вҖ” Cloudflare (handles 90% of bots):**
- Enable "Bot Fight Mode" (free tier)
- Enable "Browser Integrity Check"
- Add WAF custom rule: block if `cf.client.bot = true AND uri.path starts_with "/api/search"`
- Cache product detail pages aggressively at edge

**Layer 2 вҖ” Express middleware (handles legitimate-looking abuse):**

```ts
// Sliding window in Redis
const searchLimiter = rateLimit({
  store: new RedisStore({ sendCommand: (...args) => redis.call(...args) }),
  windowMs: 60_000,
  max: 30,                          // 30 search req/min/IP
  keyGenerator: (req) => req.ip || 'unknown',
  message: { error: 'rate_limited' },
});

const browseLimiter = rateLimit({ /* ... */ max: 120 });
```

Per-route buckets:
- `/api/search` вҖ” 30 req/min/IP
- `/api/products/:slug` вҖ” 120 req/min/IP
- `/api/checkout` вҖ” 10 req/min/IP
- `/api/images/upload` (admin) вҖ” separate bucket, requires bearer token

**Layer 3 вҖ” Bot challenge on suspicious behavior:**

Cloudflare Turnstile challenge fires when:
- Same IP hits search > 200x in 5 min
- User-agent matches known scraper patterns
- Missing referer + missing accept-language headers

Pseudocode:
```ts
const suspicionScore = computeSuspicion(req);  // 0-100
if (suspicionScore > 60) return forceTurnstile(res);
```

**Layer 4 вҖ” Honeypot fields** in search forms вҖ” bots that auto-fill all fields get instant 403.

**Don't build bot detection from scratch.** Cloudflare's free tier blocks the cheap bots; Turnstile blocks the medium ones; the rest (motivated scrapers) get rate-limited into uselessness.

## Tax design

**Schema:** `tax_rates` table seeded once with state + base rate. Most states have additional county/city rates (e.g. CA Los Angeles County adds 2.25%). For MVP, base state rate is "good enough"; switch to TaxJar/Avalara API when you have nexus in 5+ states or revenue > $100K/yr.

**Lookup at checkout:**
```sql
SELECT rate FROM tax_rates WHERE state_code = $1;
```

**Calculation:**
```ts
const taxCents = Math.round(subtotalCents * Number(rate));
```

Always integer arithmetic вҖ” never floats on money.

**Out-of-state:** charge 0% tax for states where you don't have nexus. Track nexus separately:

```sql
CREATE TABLE tax_nexus (
  state_code  CHAR(2) PRIMARY KEY,
  active_from DATE NOT NULL,
  reason      TEXT  -- 'physical_presence', 'economic_threshold', 'voluntary'
);
```

If `state_code` not in `tax_nexus`, no tax. South Dakota v. Wayfair economic nexus: $100K revenue or 200 transactions per state per year typically triggers nexus.

## S3 image upload pipeline

**Flow:**
1. Admin uploads original image to API (POST `/api/admin/products/:id/images`, multipart)
2. API streams to S3 at `products/{id}/original.{ext}`
3. Background job (BullMQ on Redis) generates 3 sizes: thumb (200), medium (600), large (1200), all as WebP + AVIF
4. Update `product_images` with all 6 s3_keys (3 sizes Г— 2 formats)
5. CloudFront invalidates the product's path

**Cost estimate:** at 50K products Г— 6 variants Г— ~50KB avg = 15 GB. S3 + CloudFront: ~$5/month for storage + bandwidth at modest scale.
